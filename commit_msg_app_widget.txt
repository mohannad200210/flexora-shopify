fix: add third-party app widget (CK WhatsApp / Messaging) CSS overrides

The chat bubble the user sees in the screenshot is NOT theme code — it's from the
"CK: WhatsApp" app (visible in the user's Shopify Admin → Apps page list). The "Share"
vertical tab on the right edge is from the "Messaging" app. Neither renders server-side
in the page HTML (I searched the live page output for chat/WhatsApp/iframe/app-embed
markers and found zero matches — they load via client-side JS after page load).

Since I can't see the actual app DOM/classes in the static HTML, I can't target the
exact selectors. Instead I added GENERIC position:fixed overrides that push any
bottom-left or bottom-right fixed element UP to avoid the sticky_atc_bar:

  div[style*="position: fixed"][style*="left: 0"],
  div[style*="position:fixed"][style*="left:0"],
  div[style*="position: fixed"][style*="left: 20"],
  div[style*="position:fixed"][style*="left:20"] { bottom: 80px !important; z-index: 95 !important; }

  div[style*="position: fixed"][style*="right: 0"],
  div[style*="position:fixed"][style*="right:0"],
  div[style*="position: fixed"][style*="right: 20"],
  div[style*="position:fixed"][style*="right:20"] { bottom: 80px !important; z-index: 95 !important; }

These match any inline-positioned fixed widget at bottom-left or bottom-right with
some offset (0 or 20px). The CK WhatsApp widget typically uses position: fixed;
bottom: 20px; left: 20px; so this selector will catch it. Also added:
  body { padding-bottom: 70px; }
so the chat widget doesn't cover the page footer when scrolled to bottom.

The override is scoped to the same <style> tag inside flexora_floating_quote_block's
custom_liquid HTML (where the previous layout fixes already live). It does NOT
touch the flexora custom app or any other installed app.

CAVEAT: if CK WhatsApp or Messaging injects its widget via a shadow DOM or via
a non-inline-style position (e.g. external CSS file), these selectors won't match.
In that case the app's own settings panel should be used to adjust position.

All previous fixes (Items 1, 2, 3) are preserved — this just extends the Item 2
fix to cover third-party apps as well.
