fix: move flexora_features + flexora_floating_quote from top-level sections to main-product blocks

Reason for change: this theme's main-product section has a fixed block schema and a fixed block_order array. Custom full sections can ONLY be added at the top-level data.order, which renders AFTER main-product ends (after description, collapsible tabs, sticky_atc, etc.). That put flexora_features and flexora_floating_quote visually at the bottom of the product page — wrong per the original ask.

Fix: convert them from custom-liquid TOP-LEVEL SECTIONS to custom-liquid BLOCKS inside sections.main.blocks, then place them at the right block_order positions:
- flexora_floating_quote_block: position 1 (right after title — the "near the very top of the page" the user asked for)
- flexora_features_block: position 4 (right after price — the "right after the hero/price area" the user asked for)

This also fixes Item 2 (reposition existing checkmark benefit bullets) — the product-specific bullets (dual lumbar support, etc.) are now in the icon grid (flexora_features_block) above the variant picker.

flexora_floating_quote_block details (per Issue 2):
- Original implementation was a static in-flow block (no position: fixed/sticky)
- Now uses position: sticky; top: 8px; z-index: 5 — stays at the top of the viewport when scrolled to, scrolls naturally out of view
- Smaller design (compact horizontal chip with quote + name + (مثال) marker + stars)
- Did NOT use position: fixed because it would obstruct the sticky_atc bar (Item 5) and overlap with the cart drawer; sticky-positioning is the practical equivalent of "floating" without those UX issues

flexora_features_block details:
- Inline 3-col icon grid (no longer uses the icon-bar section — just inline HTML, fits better in a block context)
- 6 product-specific features with Material Symbols icons + short Arabic title + 1-line description
- Background: teal-tinted gradient (matches site color scheme)

Also affects: top-level order is now main → flexora_story_posture → ... (no more flexora_features or flexora_floating_quote as top-level sections).
